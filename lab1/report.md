# Обработка данных FASTQ


## 1. Подсчёт числа прочтений и длин (`fastq_parser`)

Для работы с прочтениями использовалась утилита `awk`.

В формате FASTQ каждая запись занимает 4 строки, поэтому обрабатывались только строки последовательностей (`NR % 4 == 2`). Для каждой последовательности вычислялась её длина, после чего обновлялись искомые данные.


## 2. Подсчёт GC-состава (`gc_counter`)

GC-состав также вычислялся с помощью `awk`.

Для каждой последовательности считались:
- общее число нуклеотидов,
- количество символов `G` и `C`.

После этого вычислялся процент GC.

## 3. Среднее качество для позиции 10 (`phred_counter`)

Для вычисления был использован Python-скрипт.

Файл FASTQ читается построчно с помощью `with open(...)`.

Строка качеств берется по условию `i % 4 == 3`. Из этих строк выбирается символ на 10-й позиции. Если длина строки достаточна, символ качества переводится в числовое значение **Phred+33**

После завершения чтения файла вычисляется среднее значение качества для 10-й позиции.

## 4. Тримминг по качеству (`trimmo`)

Для этого задания использовался Python-скрипт.

Функции:

- `read_fastq(path)` — генератор для последовательного чтения FASTQ-файла по 4 строки. Файл обрабатывается постепенно, без полной загрузки в память (`yield`).
- `trim_by_quality(seq, qual, window, lim)` — функция для тримминга по качеству. Она переводит строку качеств в список значений **Phred+33**, затем просматривает окнами длины 5. Если среднее качество окна не превышает 30, прочтение обрезается от начала этого окна.

После **quality trimming**:

- подсчитывается количество прочтений, полностью удалённых в результате тримминга по качеству;
- для непустых прочтений определяем минимальную, среднюю и максимальную длину.

Затем фильтр по длине:

- оставляем только прочтения длиной от 60;
- подсчитываем число оставшихся прочтений.
